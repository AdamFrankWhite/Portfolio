Declarative Programming - a very useful concept

Declarative Programming is all about making life easier for all programmers. Remember, you aren't going to be the only one looking at your code, so it needs to be easy to understand. We all know after a couple of hours of heaving programming, looking at a jumbled, cryptic mess of code is exhausting, compared to code that's easy to read and understand.

Declarative programming is about making the code speak for itself.Let's take a simple example.Here is a code snippet hiding an error message:

$('#error').css({'visibility':'hidden'});

Yes, in this simple example, understanding this simple JQuery method isn't too difficult.But what about this piece of code?

hideError();

We can't get much simpler than that, can we? By placing our code in its own function, not only do we make our code more maintainable, but we simplify the reading of our code.

Imagine if your code was ten times more complicated, declarative programming would make the world of difference in reading an understanding more complex code.

Developer Tools - your greatest ally

When it comes to designing web pages, Developer Tools (aka DevTools) is a powerful tool to help you design web pages and apps, as well as helping you make responsive sites. In Chrome, on the far left of the tool bar, there are two buttons. The first toggles the element selector, useful for navigating your page's DOM, helping you to visualise padding and margins.

Next to this button is the device toolbar, which helps you to visualise your design on different devices and screen sizes. Whilst ideally you would test your site on different sized devices in real-life, nonetheless, this gives you are good idea of how your site will looks on different sized screens. Also, it's unlikely you'll have every single mobile and tablet device laying around to test manually!

Getting Organised - using Trello or Jira

When designing a site or app, there are always so many things to do, and without adequate organisation, it is easy to get lost, overwhelmed, and make mistakes - mistakes that could cause a lot of hassle further down the line.

This is where organisational tools like Trello and Jira come into play. Whilst Jira is specifically meant for developers to use, Trello is also really useful. By breaking your project up into small tasks, you don't have to waste time and precious mental resources remembering what you need to do, or forgetting it!

When I first started using Trello to organise my work, the benefit was hugely noticeable. My stress went down and my productivity went up.

Bootstrap - to use or not to use....

Bootstrap is a front-end framework, designed by the folks behind Twitter. It is a useful tool, providing simple shortcuts and code snippets to help speed up your designing, and provide uniformity. However, it does have its drawbacks.

Some designers swear by it, others swear off it. As for me, whilst at first I found it relaly useful - its speed and ease is great, unfortunately I found that once I wanted to customise my designs more and tweak my own custom css stylesheet, it became a pain having to create super-specific selectors to take precedence over Bootstrap's own stylesheet.
