Declarative Programming vs Imperative programming

Declarative Programming is all about making life easier for programmers.

Here's the difference between the two:

  • Declarative programming is code that describes what you are doing.
  • Imperative programming is code that describes how you are doing something.

Declarative program takes up fewer lines of code and is easier to read - because the code describes itself. Conversely, imperative programming takes up more lines of code, and is harder to read at a glance, because you have to decipher what it is that particulary code block is doing. Wouldn't it be easier if the code could speak for itself? Enter, declarative programming.

read more...

Declarative programming has become increasinly popular as a method of programming, which is evident from the prevalance of libraries such as React and jQuery, as well as JavaScript's map, filter and reduce methods. Let's look at an example.

Remember, you aren't going to be the only one looking at your code, so it needs to be easy to understand. We all know after a couple of hours of heaving programming, looking at a jumbled, cryptic mess of code is exhausting, compared to code that's easy to read and understand.

Declarative programming is about making the code speak for itself. Let's look at an example. Say you had an array of integers that you wanted to add together:

let myArray = [3, 5, 6, 3, 7, 9];

Traditionally, we would solve this problem imperatively, by looping through the array, cumulatively adding each item to a count variable:

let myArray = [3, 5, 6, 3, 7, 9]; let total = 0; for (let i=0; i<myArray.length; i++) { total += myArray[i] } console.log(total) // 33

Our code does the job, but it's not that easy to read. Imagine if our program was 1000 times more complicated - we'd spend a lot of time and brainpower trying to understand what our code does.

What about a more declarative approach? If we use the reduce method, our code for the above problem would look like this:

let myArray = [3, 5, 6, 3, 7, 9]; let total = myArray.reduce( (total, num) => total + num, 0); console.log(total) // 33

In this example, the reduce method takes two arguments - num (each item) and sum (the running total), which is initialized to 0 at the end of the method. It then iterates through myArray, applying our code to each one. Notice though, that this is all done under the hood, behind the scenes. By abstracting the details away, our code is becomes easier to deal with. As you can see, not only does this take fewer lines of code, but it's easier to understand - we can clearly see we are reducing our array to its sum.

Now, imagine if your code was a thousand times more complicated - declarative programming would make the task much more managable.

The Wonder that is StackOverflow...

If there were wonders of the internet-world, Wikipedia would probably be one. If there were wonders of the programming world, StackOverflow would be certainly be one. Like the Oracle of Delphi, if you ask StackOverflow a programming question, fellow programmers will respond with a wise answer (usually several, ranked in order of votes).

read more...

As a programmer, you'd better get used to getting stuck. If you become good at identifying your problem and formulating an appropriate question (using the correct terminology), you can usually figure most things out. Google your problem and you will almost certainly find someone else having encountered the same problem, along with excellent advice on how to solve it. If you can't find it, you can then ask fellow programmers on StackOverflow directly.

Here are some random recent searches I have made:

  • "jQuery parent() does not work"
  • "javascript set object key with variable"
  • "adding event listeners to dynamically created element javascript"
  • "how to float nav items right but keep order"
  • "accessing ajax response data"
  • "highlighting menu item when scrolling javascript"
  • "parseerror adjacent JSX elements must be wrapped in an enclosing tag"
  • "XMLHttpRequest blocked by CORS policy"
  • "addeventlistener in for loop always adds last item"

As you can see, so long as you are precise in your query, you generally find a solution. I have learned so much from reading and absorbing the advice of other programmers. By connecting programmers of all types and all of their problems, StackOverflow is a powerful resource - without it, learning to programming would undoubtedly be several orders of magnitude more difficult.

The humble beginnings of a programmer...

Looking for a career-change, I decided to teach myself the basics of front-end programming. Whilst my original introduction to programming was creating a simple HTML website when I was 16, several years later, I found myself drawn back into programming.

read more...

This time around, my first proper introduction into programming was with Python. Starting from scratch, with Team Treehouse holding my hand, I began to learn about variables, functions, loops and conditional statements. At this stage, knowing absolutely nothing, learning how to think programmatically learning took some time.

I could soon follow along reasonably well, and could tweak scaffolded code, although when placed in front of a blank editor, I was completely lost. It was at this point, eager to move into web development, I decided to focus on HTML, CSS, and JavaScript.

Initially confused at a change in language, I soon became more comfortable reading and writing code. Following advice I had often heard, I began to build my own 'projects'. Using freecodecamp as a launching pad, as well as Team Treehouse, I began to create my first div containers, floated elements, stylesheets and, eventually, basic web pages. and CSS.

As time went by, and after more practice, things began to make more sense. Whilst I initially felt overwhelmed by the range of languages, concepts, frameworks and libraries, as well as the sheer amount that I didn't know (it took me quite a while to get my head around exactly a library was), bit bit by things started falling into place.

The most crucial skill I learned was the skill of learning itself. As I became more familiar with my chosen languages, my rate of learning began to improve and the speed at which I picked up new concepts increased. That feeling of being overwhelmed was replaced with a sense of curiousity and confidence that I could figure out whatever I needed to, eventually.

I slowly began to think like a programmer, and I found myself seeing conditional statements everywhere, both in digital technologies, games, and in real-life. I found myself waking up with thoughts of how to better approach a problem in my current project. This is when I realised I had found the career for me.

Looking back, when I first started the never-ending acronyms and abstract technologies were intimidating. Nonetheless, I gradually figured out a path of frameworks and libraries that I wanted to learn. I became increasingly more comfortable not knowing everything, and the learning became addictive. Torn between wanting to learn everything, and needing to focus, I slowly absorbed the basics of a variety of tools including JQuery, AJAX, and Bootstrap. When I then starting learning about each one, I reminded myself that not so long ago, they were just abstract names, and that I now had a working knowledge of them. I understand now that never-ending learning is part of the charm of programming.

What's so special about JQuery?

Personal note - this is an exciting post. To know enough about JavaScript to be able to appreciate the utility of JQuery.

JQuery is one of the most popular JavaScript libraries. It serves two main purposes:

1) JQuery provides extensive cross-browser support.

Cross-browswer support may not be glamorous, but taking out the hassle of making sure your code runs on different broswers is very useful.

2) It simplifies so much JavaScript code with shorthand methods.

These shorthand methods help simplify DOM traversal and manipulation, event handling, AJAX, and animation.

read more...

Take, for example, an AJAX request. In vanilla (plain) JavaScript, it would take this many lines of code to make a request:

var request = new XMLHttpRequest(); request.open('GET', 'someaddress.com/file.json'); request.onload = function () { var results = JSON.parse(request.responseText); //do something with results; } request.send();

However, with jQuery, an AJAX request is more streamlined:

var $request = $.ajax({ url: "http://www.json-generator.com/api/json/get/coSwXiWUky?indent=2", success: function(result){ data = result; // assign results to variable // do stuff with result } });

Rather that creating a XMLHttpRequest object, then opening a request, adding a callback function, then sending the question, with JQuery, we supply the data as an obect, and JQuery does the rest!

As for compatibility, as well as checking caniuse.com for whatever property or feature you are trying to implement, jQuery offers well-supported cross-browser compability, and should be taken full advantage of.

Dev Tools - your greatest ally

When it comes to designing web pages, Developer Tools (aka DevTools) is a powerful tool to help you design web pages and apps, as well as helping you make responsive sites. In Chrome, on the far left of the tool bar, there are two buttons. The first toggles the element selector, useful for navigating your page's DOM, helping you to visualise padding and margins.

Next to this button is the device toolbar, which helps you to visualise your design on different devices and screen sizes. Whilst ideally you would test your site on different sized devices in real-life, nonetheless, this gives you are good idea of how your site will looks on different sized screens. Also, it's unlikely you'll have every single mobile and tablet device laying around to test manually!
